1. PROJEYE GENEL BAKIS
================================================================================

proje001, kurumsal duzey bir Yapay Zeka Orkestrasyon Platformudur.
Temel amaci; sirket belgelerini, ses/video kayitlarini, sunumlari ve Excel
dosyalarini anlamli bilgiye donusturek calisanlara dogal dil arabirimi
(sohbet) uzerinden kurumsal hafizaya erisim saglamaktir.

TEMEL YETENEKLER
----------------
Kurumsal RAG          : Sirket belgelerine semantik arama ve LLM yaniti
Ses Transkripsiyon    : Toplanti kayitlarini metne cevirmek (Whisper)
Cok Formatli Arsiv    : PDF, PPTX, Excel, Word, BPMN, MP3, MP4 isleme
Cok Model Destegi     : OpenAI, Google Gemini, Anthropic Claude, Ollama
Erisim Kontrolu       : Belge bazinda rol/politika yonetimi
Ajan Orkestrasyonu    : Farkli gorevler icin ozellestirilmis AI ajanlar
Canli Izleme          : Token tuketimi, maliyet, oturum analitigi
N8N Otomasyonu        : Is akisi otomasyonu entegrasyonu
Bilgi Grafigi         : NetworkX tabanli chunk iliski haritasi

7. TEMEL SERVISLER VE IS MANTIGI
================================================================================

7.1 AIService (backend/services/ai_service.py)
-----------------------------------------------
Tum yapay zeka etkilesimlerinin merkezi.

get_reply() -- Tam yanit modu:
  1. Kullanici mesaji alinir
  2. Kullaniciya ait haric tutulan dosyalar meta'dan okunur
  3. pgvector'de semantik benzerlik aramasi yapilir (Top-K aday)
  4. Kosinus uzakligi filtresi uygulanir (<= 1.6)
  5. Bilgi grafiginde komsu chunk'lar genisletilir
  6. Belgeler tablosuyla JOIN yapilarak zengin bagiam olusturulur
  7. Erisim politikasi filtresi uygulanir
  8. Ses/video havuzu (rag_2) tespiti yapilir
  9. Bagiam sinirlandirilir ve prompt olusturulur
 10. LLM cagrilir (ilgili saglayiciya gore)
 11. Yanit + RAG kaynaklari + UI aksiyonu dondurulur
 12. Mesaj oturuma kaydedilir, log olusturulur

get_reply_stream() -- SSE akisli mod:
  - Ayni adimlar, ancak LLM yaniti token bazinda akitilir
  - Frontend EventSource ile anlik gosterir

revise_prompt() -- Prompt duzeltici:
  - Ayri bir bot ile kullanici promptunu netlestir
  - Belirsiz sorulari kurumsal bagama gore zenginlestir

7.2 Memory Service (backend/services/memory.py)
------------------------------------------------
  - Her sohbet oturumu icin ayri pgvector koleksiyonu: chat_mem_<session_id>
  - Yeni sorguda semantik geri cagirma ile ilgili gecmis mesajlar getirilir
  - Hem RAG bagiam hem sohbet hafizasi LLM'e birlikte sunulur
  - Halusinasyonu azaltir

7.3 Monitor Service (backend/services/monitor_service.py)
----------------------------------------------------------
  - Her API cagrisinda token sayisi ve maliyet hesaplanir
  - Model ve saglayici bazinda istatistik tutulur
  - Gercek zamanli pano icin aggregasyon yapar
  - Oturum sure analizi saglar

7.4 Processor Dispatcher (backend/services/processor.py)
---------------------------------------------------------
  - Dosya uzantisina gore dogru isleyiciyi yonlendirir
  - PPTX -> PDF donusumunu yonetir
  - Chunk'lama ve vektorlestirme koordinasyonu
  - Atomik transaction yonetimi

      |

      |

      |

8.2 EMBEDDING MODELI
---------------------
Model   : all-MiniLM-L6-v2 (sentence-transformers)
Boyut   : 384
Ozellik : Cok dilli, hizli, hafiza dostu
Depolama: PostgreSQL vector(384) sutunu (pgvector)

8.3 BILGI GRAFIGI
------------------
NetworkX tabanli graf yapisi, chunk'lar arasi iliskileri saklar:

  next_chunk    : Sirali metin akisi
  semantic      : Anlam benzerligi
  visual_detail : Gorsel detay baglantisi
  action_flow   : Is akisi iliskisi

RAG sorgusunda bulunan chunk'larin 1-hop komsulari otomatik eklenerek
bagiam zenginlestirilir.

8.4 ERISIM POLITIKALARI
------------------------
Belge duzeyinde politika sistemi:

  public       : Tum kullanicilar
  hr           : IK yetkisi olan
  finance      : Finans yetkisi olan
  admin        : Yoneticiler
  confidential : Ozel erisim

10.4 GENEL ISLEM AKISI
------------------------
POST /bridge/upload-and-analyze
    |
    |-> Dosyayi temp_uploads/<uuid>_<dosyaadi> konumuna kaydet
    |-> ThreadPool'da islemeyi baslat (blocking olmayan)
    |       |-> Uzantiya gore processor sec
    |       |-> Chunk'lari uret
    |       |-> Embedding uret (all-MiniLM-L6-v2)
    |       |-> pgvector'e ekle
    |       |-> Bilgi grafigi kenarlari olustur
    |       `-> belgeler + vektor_parcalari tablosunu guncelle
    `-> task_id ile don (frontend progress icin poll eder)

10.5 ATOMIK CHUNK SILME
-------------------------
DELETE /bridge/chunk/{chunk_id} ozel bir islem yapar:
  1. Chunk silinir
  2. Komsu chunk'lar tespit edilir
  3. Bilgi grafigi yeniden baglanir (re-linking)
  4. Ilgili gorsel dosyasi silinir (varsa)
  5. parca_sayisi guncellenir
  6. Hepsi tek transaction icinde

13. VERI AKISLARI

13.1 SOHBET VE RAG AKISI
--------------------------
Kullanici -> POST /chat/stream
    |
    |-- Kullanici meta'dan haric tutulan dosyalar okunur
    |-- pgvector: cosine similarity arama (Top-K)
    |-- Uzaklik filtresi (<= 1.6)
    |-- NetworkX: 1-hop komsu genisletmesi
    |-- SQL JOIN: belge meta, erisim politikasi
    |-- Erisim filtresi (kullanici izinleri)
    |-- Havuz tespiti (rag_1 belge / rag_2 ses-video)
    |-- Bagiam sinirlandirma (token limiti)
    |-- LLM prompt insasi
    |-- LLM -> SSE akisli yanit
    |-- Mesaj oturuma kaydedilir
    `-- Monitor log olusturulur

13.2 DOSYA YUKLEME AKISI
--------------------------
Kullanici -> POST /bridge/upload-and-analyze
    |
    |-- Dosya temp_uploads/ konumuna UUID prefix ile kaydedilir
    |-- ThreadPool'da asenkron isleme baslar
    |       |-- Uzantiya gore processor secilir
    |       |-- Icerik cikarilir (metin/tablo/ses/gorsel)
    |       |-- Chunk'lara bolunur
    |       |-- all-MiniLM-L6-v2 ile embedding uretilir
    |       |-- pgvector'e kaydedilir
    |       |-- Bilgi grafigi kenarlari olusturulur
    |       `-- belgeler tablosu guncellenir
    `-- task_id doner -> Frontend GLOBAL_PROGRESS'i poll eder

13.3 OTURUM HAFIZASI AKISI
---------------------------
Yeni Sorgu
    |
    |-- pgvector koleksiyon: chat_mem_<session_id>
    |-- Semantik geri cagirma: ilgili gecmis mesajlar
    |-- Son N mesaj: sohbet_mesajlari tablosundan
    |-- RAG bagiami + sohbet hafizasi birlestir
    `-- LLM'e tek prompt olarak gonder
